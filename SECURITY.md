# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Current live version (kiezquiz.de) | ✅ |
| Older versions | ❌ |

## Sicherheitslücke melden / Reporting a Vulnerability

Bitte melde Sicherheitslücken **NICHT** als öffentliches GitHub Issue.

Verwende stattdessen einen dieser Meldewege:

1. **GitHub Private Vulnerability Reporting:** Klicke im Repository-Tab „Security“ auf „Report a vulnerability“
2. **E-Mail:** [info@kiezquiz.de](mailto:info@kiezquiz.de) (PGP-verschlüsselt auf Anfrage)

## Reaktionszeiten

| Phase | Zeitrahmen |
|-------|------------|
| Eingangsbestätigung | 72 Stunden |
| Erstbewertung | 14 Tage |
| Behebungsplan | 90 Tage |

## Scope

**In Scope:**

- kiezquiz.de (Frontend/PWA)
- Supabase-Backend (RLS, Edge Functions, Auth)
- GitHub-Actions-CI/CD-Pipeline
- iOS/macOS-App (KiezQuiz-App)

**Out of Scope:**

- Social Engineering
- Physische Angriffe
- DoS/DDoS
- Spam
- Angriffe auf Drittanbieter-Infrastruktur (Supabase, GitHub, Resend)

## Danksagung / Hall of Fame

Wird bei verantwortungsvoller Meldung ergänzt.
