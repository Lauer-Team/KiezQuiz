Erinnerung: Supabase-Daten exportieren und archivieren (DSGVO / TOMs).

Betrieb & Datenschutz

Monatlicher Daten-Export

Warum? Supabase Free Tier hat keine automatischen Datenbank-Backups. Für die TOMs (Art. 32 DSGVO) sichern wir die Nutzerdaten monatlich per SQL-Export — Profile, Spielstände, Freunde, Bestenlisten usw. (Schema public).

GitHub Actions erstellt am 1. jeden Monats automatisch ein Backup. Diese E-Mail erinnert dich daran, es herunterzuladen und dauerhaft zu archivieren.

Schritt 1 — Backup prüfen & herunterladen

  1. GitHub → Actions → Supabase monthly backup
  2. Letzten Run öffnen — Status muss grün sein
  3. Unter Artifacts die Datei supabase-backup-… herunterladen
GitHub Actions öffnen

Schritt 2 — Dauerhaft archivieren

GitHub speichert Artifacts nur 90 Tage. Kopie ablegen in z. B. Ordner „Rechtsdokumente / KiezQuiz Backups“, iCloud Drive oder externe Festplatte.

Alternative: Manuell auf dem Mac 

python3 scripts/setup_supabase_backup.py   # einmalig
python3 scripts/export_supabase_backup.py --dry-run
python3 scripts/export_supabase_backup.py
